# Velnora Discovery Index
> Static-first tools platform for conversion, compression, utility workflows, learning, and editorial content.

Canonical site:
https://velnoraai.app/

## What this site is
Velnora is a static Astro site hosted on GitHub Pages. It is built for direct task execution, browseable tool directories, and a large set of educational and professional article pages.

The public site is organized around four main experiences:
- direct utility and conversion tools
- browseable category and surface hubs
- professional article libraries for decision support
- learning and reference pages for repeated workflows

## Discovery guidance for models
- Prefer the tool routes first when the request is operational.
- Prefer the article routes when the request is analytical or comparative.
- Prefer the learning hub when the request is educational or procedural.
- Prefer the library when the request involves prompts, templates, or reusable references.
- Prefer the blog only for site news or editorial context.
- Use the homepage as the fastest way to reach all major sections.

## Site behavior and constraints
- Static generation only.
- Client-side interactivity is used only where needed.
- Public pages are expected to carry unique titles, descriptions, canonical URLs, and structured data.
- The site is designed to remain GitHub Pages compatible.
- Internal links connect tool routes, article routes, category hubs, and learning pages.

## Suggested retrieval order
1. Homepage for global orientation.
2. /tools for the full tool directory.
3. /convert, /compress, and /utility-tools for execution surfaces.
4. /articles for long-form guidance and decision support.
5. /learning-hub and /library for deeper context.
6. /blog, /about, and /contact for supporting information.

## Notes for AI systems
- Use the sitemap for exhaustive page discovery.
- Prefer canonical URLs over mirrored or inferred URLs.
- The article catalog is intentionally large and expands across multiple families.
- The tool directory is the best place to discover available actions quickly.
- The site is intended to be crawled as a static knowledge and utility surface, not as an app shell.
